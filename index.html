📚 SC-900 개념 백과사전

제공된 120문제를 완벽 분석하여 정리한 모든 개념입니다.

🆔 Identity & Access
Azure Active Directory (Azure AD / Entra ID) 기본
Microsoft의 클라우드 기반 ID 및 액세스 관리 서비스.
👉 인증(Authentication) 및 권한 부여(Authorization)를 담당함. 테넌트 관리, 사용자 관리의 핵심.
AD DS vs Azure AD 비교
온프레미스(서버) 기반의 전통적 AD와 클라우드 기반 AD의 차이.
👉 Azure AD는 HTTP/HTTPS 통신을 사용하며, 온프레미스 AD처럼 Kerberos 인증을 주력으로 하지 않음. Azure VM을 호스팅하기 위해 필수적인 것은 아님(구독이 필요).
Hybrid Identity (하이브리드 ID) 동기화
온프레미스 AD와 Azure AD를 연결하여 하나의 ID로 양쪽 리소스에 접근하게 함.
👉 Azure AD Connect 도구를 사용하여 동기화함. (AD FS 등도 사용 가능)
MFA (Multi-Factor Authentication) 인증
두 가지 이상의 인증 요소를 요구하여 보안을 강화함. (보안 점수 상승의 핵심)
👉 요소: 지식(비밀번호), 소유(핸드폰, 앱), 생체(지문, 얼굴).
👉 Security Defaults 활성화 시 모든 사용자에게 강제됨.
SSPR (Self-Service Password Reset) 기능
사용자가 헬프데스크 없이 스스로 비밀번호를 변경/초기화.
👉 Azure AD Premium 기능. 사용자는 사전에 인증 방법(휴대폰, 이메일 등)을 등록해둬야 함.
Passwordless Authentication 인증
비밀번호 없이 로그인하는 보안 방식.
👉 Windows Hello for Business (얼굴/지문/PIN), FIDO2 보안 키, Microsoft Authenticator 앱.
Conditional Access (조건부 액세스) 보안 정책
로그인 시도 시, 신호(Signal)를 분석하여 결정(Decision)을 내리고 집행(Enforce)함.
👉 신호: 사용자/그룹, 위치(IP), 디바이스 상태, 애플리케이션, 위험(Risk).
👉 조치: 허용, 차단, MFA 요구.
👉 Azure AD Premium P1 이상 필요.
Identity Protection 위험 관리
로그인 및 사용자와 관련된 위험을 탐지하고 자동 대응함.
👉 User Risk(사용자 위험): 자격 증명 유출(Leaked Credentials).
👉 Sign-in Risk(로그인 위험): 평소와 다른 위치, 익명 IP.
👉 Azure AD Premium P2 필요.
PIM (Privileged Identity Management) 권한 관리
관리자 권한을 상시 부여하지 않고, 필요할 때만 시간 제한을 두고 부여함.
👉 JIT (Just-In-Time), Time-bound(시간 제한), 승인(Approval) 절차.
👉 Azure AD Premium P2 필요.
Identity Types (ID 유형) 개념
Azure AD 내의 다양한 ID 객체들.
👉 Service Principal: 애플리케이션이 리소스에 접근하기 위한 ID.
👉 Managed Identity: Azure 리소스(예: VM)가 비밀번호 없이 다른 Azure 서비스에 인증하기 위한 자동 관리 ID.
External Identities (외부 ID) 협업
조직 외부 사용자(Guest)를 초대하여 협업 (B2B).
👉 외부 이메일 주소로 초대 가능, SSPR 사용 가능.
🛡️ Security
Zero Trust (제로 트러스트) 원칙
"절대 신뢰하지 말고 항상 검증하라"는 보안 모델.
👉 3대 원칙: 1. 명시적 검증(Verify Explicitly) 2. 최소 권한 부여(Least Privilege) 3. 침해 가정(Assume Breach).
Shared Responsibility Model 클라우드 기초
클라우드 제공자와 고객 간의 책임 분담 모델.
👉 SaaS: 고객은 데이터, ID, 디바이스만 책임.
👉 IaaS: 고객이 OS, 네트워크, 애플리케이션 등 더 많은 부분 책임.
👉 PaaS: 그 중간.
Azure DDoS Protection 네트워크
분산 서비스 거부 공격으로부터 보호.
👉 Standard: 가상 네트워크(VNet) 리소스를 보호. AI 기반 트래픽 모니터링 및 완화.
NSG (Network Security Group) 네트워크
Azure 가상 네트워크 내의 트래픽을 필터링하는 방화벽 역할.
👉 IP 주소, 포트, 프로토콜 기반으로 인바운드/아웃바운드 트래픽 허용/차단. 서브넷이나 NIC에 연결.
Azure Bastion 네트워크
브라우저(Azure Portal)를 통해 VM에 안전하게 RDP/SSH 접속.
👉 Public IP가 필요 없음. TLS(443 포트)를 통해 안전하게 연결. VNet당 하나만 생성 가능.
Microsoft Sentinel SIEM/SOAR
클라우드 네이티브 SIEM(보안 정보 이벤트 관리) 및 SOAR(보안 오케스트레이션 자동화) 솔루션.
👉 Playbooks: Logic Apps 기반의 자동 대응(Automation).
👉 Workbooks: 데이터 시각화 및 모니터링.
👉 Hunting: 위협 사냥 (MITRE ATT&CK 기반).
👉 Connectors: 다양한 소스(AWS 등)에서 데이터 수집.
Defender for Cloud CSPM/CWPP
Azure, Hybrid 환경의 보안 태세를 관리하고 워크로드를 보호.
👉 Secure Score(보안 점수): 보안 상태를 점수화하고 권장 사항(Recommendations) 제공.
👉 Just-In-Time VM Access: 관리 포트를 필요할 때만 염.
Defender for Endpoint 디바이스
엔드포인트(PC, 모바일) 보안 플랫폼.
👉 ASR(Attack Surface Reduction): 공격 표면 감소.
👉 Automated Investigation: 자동 조사 및 복구.
👉 Windows, macOS, Linux, Android, iOS 지원.
Defender for Identity 온프레미스 AD
온프레미스 Active Directory 신호를 모니터링하여 침해 탐지.
👉 손상된 자격 증명, 횡적 이동(Lateral Movement), 정찰 활동 탐지.
Defender for Cloud Apps CASB
클라우드 액세스 보안 브로커(CASB). SaaS 앱 사용을 모니터링.
👉 Shadow IT: 승인되지 않은 앱 사용 발견 및 제어.
👉 클라우드 앱의 데이터 이동 제어.
Defender for Office 365 이메일
이메일 및 협업 도구(Teams, SharePoint)의 위협 방어.
👉 Safe Attachments/Links: 악성 첨부파일 및 링크 실시간 검사.
👉 Phishing: 피싱 공격 방어.
📋 Compliance & Governance
Sensitivity Labels (민감도 레이블) 정보 보호
데이터(문서, 이메일)를 분류하고 보호하는 태그.
👉 기능: 워터마크 표시, 헤더/푸터 추가, 암호화(Encryption) 적용.
👉 사용자가 수동 적용하거나 자동 적용 가능.
DLP (Data Loss Prevention) 정보 보호
민감한 정보(신용카드, 주민번호 등)가 외부로 유출되는 것을 방지.
👉 Policy Tip: 사용자에게 위반 알림 표시.
👉 Exchange, SharePoint, OneDrive, Teams, Endpoint 적용 가능.
Retention Policies (보존 정책) 데이터 수명
데이터를 일정 기간 동안 보존하거나, 기간이 지나면 삭제함.
👉 사용자가 삭제해도 사본을 보존할 수 있음. (eDiscovery 등을 위해)
Information Barriers (정보 장벽) 내부 통제
이해 상충 관계에 있는 부서 간의 통신 및 데이터 공유를 차단.
👉 예: 금융권의 트레이딩 부서와 리서치 부서 간 Teams/Exchange 차단.
eDiscovery (전자 증거 개시) 법적 대응
법적 소송 시 증거로 사용할 전자 데이터를 식별, 보존, 수집.
👉 Content Search: 키워드로 데이터 검색.
👉 Hold (보존): 데이터가 삭제/수정되지 않도록 잠금.
👉 Core vs Advanced: Advanced는 더 심층적인 분석 및 변호사 검토 지원.
Insider Risk Management 내부 위협
조직 내부자(직원)에 의한 악의적 행위나 데이터 유출 탐지.
👉 퇴사 예정자의 대량 다운로드, 기밀 유출, 직장 내 괴롭힘 등 탐지.
Communication Compliance 내부 통제
부적절한 메시지(욕설, 성희롱, 협박 등)를 탐지하고 규정 준수 여부 확인.
Compliance Manager 관리 도구
조직의 규정 준수 상태를 평가하고 관리하는 도구.
👉 Compliance Score: 준수 점수 제공.
👉 Assessment Template: GDPR, ISO 등 규정 템플릿 제공.
👉 Improvement Actions: 예방(Preventative), 탐지(Detective), 교정(Corrective) 조치 제안.
Audit (감사) 로그
사용자 및 관리자 활동 기록.
👉 Unified Audit Log: 다양한 서비스의 로그 통합.
👉 Advanced Audit: 로그 보존 기간 연장(1년), 중요 이벤트(MailItemsAccessed 등) 정밀 감사, 고대역폭 접근.
Azure Policy 거버넌스
Azure 리소스에 대한 규칙을 정의하고 강제함.
👉 예: "특정 리전(Korea Central)에만 리소스 생성 허용", "특정 태그 필수".
👉 실시간으로 생성 차단하거나, 기존 리소스의 비준수 여부 감사.
Azure Blueprints 거버넌스
클라우드 환경(리소스 그룹, 정책, 역할 등)의 표준 세트를 정의하여 배포.
👉 환경 설정을 패키지화하여 반복적으로 배포 가능 (ISO 인증 환경 등).
Resource Locks (리소스 잠금) 거버넌스
중요한 리소스가 실수로 삭제되거나 수정되는 것을 방지.
👉 CanNotDelete: 읽기/수정 가능, 삭제 불가.
👉 ReadOnly: 읽기만 가능.
👉 상속됨 (리소스 그룹에 걸면 내부 리소스 다 걸림).
Service Trust Portal 자료실
Microsoft의 보안, 개인정보처리방침, 규정 준수 관련 문서 제공.
👉 Audit Reports: MS가 받은 제3자 감사 보고서(ISO, SOC 등) 다운로드 가능.
Privacy Principles (프라이버시 원칙) MS 정책
마이크로소프트의 개인정보 보호 원칙.
👉 Control(통제권): 고객이 데이터 주인.
👉 Transparency(투명성): 데이터 처리 과정 공개.
👉 광고 목적으로 고객 데이터 사용 안 함.
Customer Lockbox 접근 제어
MS 엔지니어가 지원을 위해 고객 데이터에 접근해야 할 때, 고객의 명시적 승인을 받게 함.
